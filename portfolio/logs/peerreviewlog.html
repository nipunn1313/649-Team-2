This table has an entry for each peer review event that you perform during the semester. Add new entries to the bottom of the table (you may need to add new rows). Leave all the previous entries in place so that at the end of the project you have a complete list of peer reviews that your group has performed. The meanign of the table entries are as follows:

  • Review number - unique identifier for each entry
  • Date  - the date that the review was performed
  • Artifact - the item that was reviewed (e.g., SD#4.2)
  • Reviewer - andrewID of the person who performed the peer review
  • # of defects- the number of defects found and detailed in the checklist
  • Link to Review Checklist - a link to the review checklist .xls file (peer_review.xls - a blank copy of the checklist)
