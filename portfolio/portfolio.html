Distributed Elevator Control System

Project Portfolio

18649 <Fall 2012>
Group 2 - Nipunn Koorapati (nkoorapa), Jacob Olson (jholson), Ben Clark (brclark), Nick Mazurek (nmazurek)

Portfolio Overview

  • Design
    • Architecture Diagram - The architecture diagram describes the objects present in the system, the replication of distributed objects, and the communication channels between objects.
    • Use Cases - The use case diagram describes the ways that agents in the system can interact with the elevator and shows relationships between these methods of interaction.
    • Scenarios and Sequence Diagrams - A collection of scenarios that represent specific instances of use cases and the corresponding sequence diagrams, which detail the message arcs sent during the given scenario.
    • Requirements I - System Object Descriptions and Message Dictionary - A list of the sensors and actuators in the system and a message dictionary describing the messages that are required for the various controllers.
    • Requirements II - Distributed Controller Requirements - A list of detailed specifications for each controller, containing input/output messages, constraints, behavioral requirements, and state diagrams.
  • Traceability
    • Sequence Diagrams to Requirements Traceability - A traceability matrix charting forward and backward traceability between every arc of every sequence diagram, and every behavioral requirement of every controller.
    • Requirements to Contstraints Traceability - A traceability matrix charting forward and backward traceability between the behavioral requirements of a controller and that controller's constraints.
    • Statecharts to Code Traceability - A traceability chart tracing each state diagram transition to a line of code.
  • Implementation
    • Elevator Control Package - A list of the files contained in the elevatorcontrol packages, and their descriptions.
  • Test
    • Unit Test Log - A description of each unit test and the current results of each test.
    • Unit Test Summary File - parseable list of unit test files
    • Integration Test Log - A description of each integration test and the current results of each test.
    • Integration Test Summary File - parseable list of integration test files
    • Acceptance Test Log - A description of each acceptance test and the current results of each test.
    • Fault Tolerance Test Log - A description of each fault tolerance test and the current results of each test.
  • Log Files
    • Issue Log - A list of all issues with the project (pending or fixed) and who fixed them.
    • Improvements Log - A log of all improvements that could be made to the course projects.
    • Peer Review Log - A list of peer reviews performed, with links to the appropriate peer review forms.
  • Scheduling
    • Network Schedule - An analysis of the elevator network as a whole, including a dictionary and schedulability analysis.
  • Fault Tolerance
    • FMEA - Failure modes and effects analysis.
    • FTA - Fault tree analysis.
  • Verification
    • High Level Requirements Verification Framework - An analysis of whether the elevator follows high level requirements.
  • Presentations
    • For each presentation, place presentation file in the presentations folder and create links to each one.
